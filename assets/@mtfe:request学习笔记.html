优化

既然配置项很多,querystring和postbody使用也不方便,以及retry也需要配置，不如集中处理一下。

 
xxxxxxxxxx
 
var retryDefaults = {
    retries: 0,
    minTimeout: 300,
    maxTimeout: 500,
}
function normalizeOpt(option) {
    if (typeof option === "string") {
        option.uri = option;
    } else {
        option.uri = option.uri;
    }
    if (option.retry) {
        if (typeof option.retry === "number") {
            var retryOption = Object.create(retryDefaults);
            retryOption.retries = option.retry;
            option.retry = retryOption;
        }
    }
    if (option.qs) {
        var search = url.parse(option.uri).search;
        option.uri += (search ? "&" : "?") + querystring.stringify(option.qs);
    }
    if (option.form) {
        option.body = querystring.stringify(option.form);
    }
    option.method = option.method.toUpperCase() || "GET";
}

需要留意的是此处qs的处理,兼顾了已有url上带qs，以及配置项里的qs

回顾最开始,当请求开始响应的时候,我们需要监听data事件,直到响应结束。

最后是对响应的封装。这里参考了node-fetch的设计。

对于响应体,区分了响应是空,string,stream时的操作，并以promise的形式返回。并提供了相应的方法:text()|json()|safeJson(),同时,设置了状态标识bodyUsed标识响应体是否已使用。（注意：上述三方法只能调用一次 否则由于标识的缘故会抛出异常）

对于响应头,主要是提供了get,set,getAll,append,has,delete等一些操作方法。并以数组的形式将header里的信息存储起来。
