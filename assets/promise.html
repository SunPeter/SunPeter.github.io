  2. 规范化,统一处理结果的调用方法和参数

promise.all promise.race

格式: Promise.all[task1,task2]

  • promise.all和promise.race里的task都是同步开始
  • promise.all会在所有任务都完成后触发then,而race是一旦task有1个完成就触发then,剩下的task继续执行,但对整个过程无影响（promise里无中断的操作，剩下的task必须进入Fullfiled或Rejected状态）
  • promise.all执行then方法后输出的数据顺序和task顺序一致

关于兼容

异常处理

在promise规范中，出现内部错误的promise对象最后都需要变为onRejected状态，而不会向外部抛出异常，所以一旦promsie内部没有进行错误处理，外部也无法捕获。浏览器中也会提示Uncaught (in promise) Error。

由于reject只能处理上一个promise中发生的异常,而当前resolve执行时产生的异常reject是捕获不到的。因此,需要在下一轮链式调用过程中对其进行异常捕获。 

 
xxxxxxxxxx
 
p.then(null,function(err){});  
//或者
p.catch(function(err){});

catch只是上一种then写法的简写

保险起见,最好保证在promsie链式调用结束后追加异常处理:

 
xxxxxxxxxx
 
if (typeof Promise.prototype.done === 'undefined') {
    Promise.prototype.done = function(onFulfilled, onRejected) {
        this.then(onFulfilled, onRejected).catch(function(error) {
            setTimeout(function() {
                throw error;
            }, 0);
        });
    };
}

控制promise状态的改变

因为promise最终的稳定状态只能是onFulFilled 或者 onRejected，在pending状态时then方法不会执行 所以我们可以将resolve放在一个异步操作中,通过then方法的执行来判断这个异步操作是否结束

 
xxxxxxxxxx
 
var r, d;
var p = new Promise(function(resolve) {
    d = Date.now();
    r = resolve;
});
p.then(function(){
    console.log(Date.now() - d);
});
//异步操作  
r();

如何让有先后顺序的promise依次执行

方法1: Task1().then(onFulfilled).then(Task2).then(onFulfilled)。用then来触发，缺点：task越多,promise链式调用就越长

 
xxxxxxxxxx
 
async().then(function(v){
console.log(v);
}).then(async).then(function(v){
    console.log(v);
})

方法2: 用一个数组存储每次then中的数据,同时更新promise对象

 
xxxxxxxxxx
 
function main() {
    var _results = [];
    function recordValue(value) {
        _results.push(value);
        return _results;
    }
    var tasks = [async, async];
    var promise = Promise.resolve();
    for (var i = 0; i < tasks.length; i++) {
        var task = tasks[i];
        promise = promise.then(task).then(recordValue);
    }
    return promise;
}
main().then(function(v){
    console.log(v);
})
